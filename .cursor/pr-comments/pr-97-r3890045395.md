## 🟠 Major — Novos (2 actionable)

### 1. `github_sync_active: true` persistido antes do import completar

**Arquivo**: [backend/src/services/githubService.ts:694-702](backend/src/services/githubService.ts#L694)
**Tipo**: Potential issue

`ProjectModel.updateSyncInfo` escreve `github_sync_active: true` e `last_sync_*` antes do pipeline de import/mapping concluir. Se o import falhar depois, o projeto fica marcado como ativo com dados inconsistentes.

**Fix**: mover o `updateSyncInfo` (campos `github_sync_active`, `last_sync_at`, `last_sync_sha`) para depois do import concluir com sucesso, ou implementar rollback que limpa o flag em caso de falha.

---

### 2. `extractUniqueFilePaths` adiciona rotas sem normalização

**Arquivo**: [backend/src/services/githubService.ts:450-458](backend/src/services/githubService.ts#L450)
**Tipo**: Potential issue

`block.route` é adicionado ao Set sem normalizar, causando mismatch nos lookups de `file_path` que usam `normalizeGithubFilePath`. A comparação falha para paths com capitalização ou slashes diferentes.

**Fix**: aplicar `normalizeGithubFilePath(block.route)` ao adicionar ao Set, e garantir que o mesmo helper seja usado consistentemente nos lookups.
